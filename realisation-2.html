Infrastructure Virtualisée

Création d'une infrastructure complète sous VMware

VMware vSphere Windows Server 2019 Linux Ubuntu Server Active Directory Apache MySQL

Contexte du projet

Ce projet avait pour objectif de concevoir et déployer une infrastructure virtualisée complète simulant un environnement d'entreprise. L'infrastructure devait être évolutive, sécurisée et permettre la mise en place de différents services (web, base de données, annuaire).

Durée du projet : 3 mois
Type : Projet individuel dans le cadre du BTS SIO

Objectifs pédagogiques

  • Maîtriser les concepts de virtualisation avec VMware vSphere
  • Concevoir une architecture réseau adaptée aux besoins
  • Déployer et configurer différents systèmes d'exploitation serveurs
  • Mettre en place un contrôleur de domaine Active Directory
  • Configurer des services web et bases de données
  • Documenter l'infrastructure de manière professionnelle

Réalisations techniques détaillées

1. Configuration de l'environnement VMware

  • Installation et configuration de VMware vSphere ESXi 7.0
  • Création de 5 machines virtuelles avec allocation optimisée des ressources
  • Configuration de virtual switches et port groups
  • Mise en place de snapshots avant chaque modification majeure

2. Déploiement du contrôleur de domaine (DC01)

  • Installation de Windows Server 2019 en mode Core
  • Promotion en contrôleur de domaine (domaine: entreprise.local)
  • Configuration des services DNS et DHCP
  • Création de l'arborescence OU, utilisateurs et groupes
  • Mise en place de GPO pour la sécurité (politique de mots de passe, verrouillage écran)

3. Configuration du serveur Web (WEB01)

  • Installation d'Ubuntu Server 20.04 LTS
  • Installation et configuration d'Apache2
  • Installation de PHP 7.4 et modules nécessaires
  • Mise en place d'un site web de démonstration
  • Configuration d'un virtual host pour plusieurs sites

4. Mise en place de la base de données (DB01)

  • Installation de MySQL 8.0 sous Ubuntu Server
  • Sécurisation avec mysql_secure_installation
  • Création de bases de données et utilisateurs dédiés
  • Configuration de l'accès distant depuis WEB01
  • Mise en place de sauvegardes automatiques avec cron

5. Serveur de fichiers et client de test

  • Configuration du serveur de fichiers FILE01 avec partages réseau
  • Mise en place de quotas et permissions NTFS
  • Intégration d'un client Windows 10 au domaine
  • Tests de connexion et accès aux différents services

Résultats obtenus

  • Infrastructure fonctionnelle : Les 5 serveurs communiquent correctement et tous les services sont opérationnels
  • Haute disponibilité : Snapshots configurés pour restauration rapide en cas de problème
  • Sécurité : Politique de sécurité Active Directory appliquée sur tous les postes du domaine
  • Documentation complète : 40 pages incluant schémas réseau, procédures d'installation et de maintenance
  • Optimisation des ressources : Allocation dynamique de la RAM permettant d'économiser 30% de ressources

Difficultés rencontrées

Problème 1 : Connectivité réseau entre VM

Difficulté : Les machines virtuelles ne communiquaient pas entre elles malgré une configuration réseau apparemment correcte.

Solution : Après analyse, j'ai découvert que le virtual switch était configuré en mode "privé" au lieu de "interne". Modification de la configuration réseau dans vSphere et redémarrage des VM pour appliquer les changements.

Problème 2 : Performance du serveur de base de données

Difficulté : Le serveur MySQL était très lent lors des requêtes avec le site web.

Solution : Augmentation de la RAM allouée de 2 Go à 4 Go et optimisation des paramètres MySQL (buffer pool, query cache). Amélioration des performances de 200%.
